# Next Run Briefing Template

## Purpose
This document captures how to brief story-room in one message so it can continue working to a defined finish line without repeated mid-run confirmations.

## Core Conclusion
Yes — if the goal boundary, autonomy scope, deliverable chain, and stop condition are all stated clearly in one message, story-room can usually continue working to the target state without repeated "continue" confirmations.

The key is not more context in general.
The key is locking:
- the target endpoint
- the autonomy boundary
- the stop condition
- the reporting cadence

---

## What Can Usually Be Pre-Authorized
For a season-scale run like the one just completed, these steps can usually be pre-authorized as one continuous pipeline:

1. batch full-script execution
2. later batch structural planning
3. later batch full-script execution
4. season final checkpoint
5. season assembly package
6. episode map sync pass
7. priority storyboard package
8. storyboard polish pass
9. production-facing shot deck package
10. reference-frame generation prompts
11. character likeness lock sheet

These are strong dependency-chain tasks, not high-risk branching decisions.
So if they are explicitly authorized up front, most mid-run check-ins can be skipped.

---

## What Must Be Stated Clearly Up Front

### 1. Final target layer
Define exactly how far the run should go.
Examples:
- full-script only
- season assembly
- priority storyboard
- polished storyboard
- shot deck
- image prompt package
- likeness lock sheet
- visual look-book skeleton

If the endpoint is not locked, the agent must pause at major milestones to avoid open-ended expansion.

### 2. Autonomy scope
State whether story-room may choose the pipeline order and continue without asking at each stage.
Useful instruction:

> You may choose the most reasonable pipeline order and continue autonomously. Do not stop at every stage to ask whether to continue.

### 3. Stop condition
State where automatic stopping should happen.
Useful instruction:

> Stop automatically once the listed deliverables are complete. After that, summarize only. Do not continue expanding the scope.

This is the most important protection against endless extension.

### 4. What does *not* require confirmation
Useful instruction:

> Unless there is a real contradiction, missing critical information, or a major creative branch decision, do not interrupt for step-by-step confirmation.

---

## Cases That May Still Require a Pause
Even with a good one-message brief, story-room may still need to ask if:

1. core creative direction conflicts with prior state
2. the endpoint is still ambiguous
3. there are multiple equally valid major branches and no priority was given
4. actor/look/platform/budget constraints materially change visual or structural decisions
5. further work has clearly moved into low-ROI endless refinement

---

## What To Include In A One-Message Brief
For best results, include these eight parts:

### A. Project object
- season id / project name
- whether this is a continuation of existing workspace state or a new run

### B. Starting state
Example:
- continue from current workspace state
- use existing state files as authority
- no need to re-ask for prior context unless there is a contradiction

### C. Endpoint deliverables
List the full chain you want completed.
Example:
- season full-script simulation
- season final checkpoint
- season assembly package
- episode map sync
- priority storyboard package
- storyboard polish
- production-facing shot deck
- reference-frame prompt pack
- character likeness lock sheet

### D. Autonomy permission
Example:
- you may decide the most reasonable task order
- you may run stage reviews / checkpoints / assembly autonomously
- do not stop at every milestone to ask whether to continue

### E. Stop condition
Example:
- stop automatically after likeness lock sheet
- do not continue into look-book, image generation, or AD-style shot lists unless asked

### F. Reporting cadence
Choose one clearly:
- report only at major milestones
- report after each major stage

### G. Prohibitions / boundaries
Example:
- do not expand into marketing, publicity, posters, or unrelated packages unless asked
- do not keep polishing forever
- do not repeatedly ask “continue?”

### H. Priority order
Example:
- prioritize completeness over polish
- prioritize asset closure over optional expansion
- prioritize reusable production-facing materials over speculative extras

## Copy-Paste Brief Template
Use or adapt this directly:

> Continue from the current workspace and existing state files; do not repeatedly ask me for prior context unless you detect a real contradiction.
> 
> You may choose the most reasonable pipeline order and continue autonomously.
> 
> Complete the following chain:
> 1. season full-script simulation
> 2. season final checkpoint
> 3. season assembly package
> 4. episode map sync
> 5. priority storyboard package (ep01 / ep10 / ep18 / ep20)
> 6. storyboard polish pass
> 7. production-facing shot deck package
> 8. reference-frame generation prompts
> 9. character likeness lock sheet
> 
> Rules:
> - Do not stop at each stage to ask whether to continue.
> - Only interrupt if there is critical missing information, a real conflict, or a major direction split.
> - You may run checkpoints, reviews, sync passes, and assembly steps on your own.
> - Stop automatically after step 9 and summarize.
> - Do not expand into further packages unless explicitly asked.
> - Prioritize complete, reusable assets over endless polish.
> - Report at major milestones only.

## Short Version
If only two things are fixed clearly up front, the run gets much smoother:

1. lock the finish line
2. lock the automatic stop condition

Most of the repeated confirmations in the previous run happened because those two were not fully fixed at the beginning.

---

## Final Practical Note
The real missing piece is usually not story context.
It is target boundary.

When the endpoint and stop rule are locked, story-room can usually carry the rest of the pipeline continuously.
