# task-entry.md

## Standard task intake template

When a user gives a new story job, normalize it into this structure:

- Request type:
- Target deliverable:
- Genre flavor:
- Protagonist:
- Starting disadvantage:
- Hidden strength / comeback fantasy:
- Core antagonist pressure:
- Opening hook target:
- Audience suspense question:
- First slap-back promise:
- Emotional payoff promise:
- Episode-end cliffhanger target:
- Tone:
- Episode length / scope:
- Hard constraints:
- Must-have moments:

If the user did not specify enough detail, infer only what is safe and mark assumptions clearly.

## Hook-first intake rule
Before dispatching any child agent, make sure the brief is clear on these six items:
- what grabs attention immediately
- what humiliation or pressure is active
- what question keeps the audience watching
- where the first reversal lands
- what emotional payoff is promised
- what cliffhanger closes the episode

If any of the six is missing, either infer cautiously and mark it as an assumption or ask for clarification when the missing point is decisive.

## Default assumptions for urban comeback short drama
- fast hook
- sharp humiliation setup
- hidden power / resource / identity / competence reveal
- escalating face-slapping payoff
- emotionally legible dialogue
- strong episode-end cliffhanger
