# comic-pipeline.md

## story-room default workflow

Use a **hook-driven orchestration loop** for most requests.

### Phase 1 - Task framing
Orchestrator identifies:
- target format
- episode count or length
- target audience mood
- opening hook target
- core suspense question
- key reversal fantasy
- required deliverable type

Build a compact intake brief that includes:
- protagonist pressure / humiliation setup
- audience question
- first slap-back promise
- emotional payoff promise
- episode-end cliffhanger target

### Phase 2 - Parallel hook generation
Spawn in parallel:

#### plot-agent
Produce:
- logline
- premise
- central conflict
- beat outline
- key reversals
- **hook map**
  - opening hook
  - suspense question
  - first reversal
  - payoff turn
  - episode-end cliffhanger
- risks

#### character-agent
Produce:
- protagonist sheet
- antagonist sheet
- supporting cast sheet
- relationship map
- voice cues
- contradiction points
- **character hook points**
  - sympathy trigger
  - anger trigger
  - desire / comeback trigger
  - emotional pivot points

### Phase 3 - Hook board merge
Before drafting scenes, orchestrator merges plot + character outputs into one **episode hook board**.

The hook board must lock:
- opening hook
- protagonist pressure point
- audience question
- first reversal
- emotional payoff beat
- episode-end cliffhanger
- role-specific risks to watch

Do not send script-agent a vague "write the episode" prompt. Send the hook board plus the current outline and character package.

### Phase 4 - Script drafting by hook board
Spawn script-agent using the hook board.

script-agent must produce:
- first-pass scene script
- dialogue rhythm
- key dramatic beats
- weak-point notes
- per-scene structure:
  - scene hook
  - scene goal
  - conflict escalation
  - turn / reversal
  - exit hook

Every scene should change something and end with forward pull.

### Phase 5 - Review
Spawn review-agent with summaries from all branches plus the hook board.
Review-agent must output:
- strengths
- **hook diagnosis**
  - opening hook
  - mid-episode retention
  - cliffhanger strength
- **logic diagnosis**
  - character motive clarity
  - reveal order
  - reversal credibility
- **payoff diagnosis**
  - humiliation clarity
  - slap-back concentration
  - emotional payoff delivery
- must-fix issues
- optional improvements
- verdict: ship / revise

Must-fix items must be routed by owner where possible:
- to plot-agent
- to character-agent
- to script-agent

### Phase 6 - Focused revision
If revise:
- send only must-fix items to the responsible branch
- do not restart the entire room unless the premise itself is broken
- ask for finite revisions only
- patch the hook board first if the hook logic changed

### Phase 7 - Final assembly
Orchestrator combines:
- final outline
- final character package
- final hook board
- final script pages
- short review summary

## Delivery modes

### Mode A - concept package
Deliver:
- title options
- logline
- premise
- cast
- outline

### Mode B - pilot episode package
Deliver:
- title
- episode synopsis
- full scene script
- cliffhanger

### Mode C - rewrite pass
Deliver:
- revised scenes only
- change log
- remaining risks

## Iteration limits
Default:
- Draft 1
- Revision 1
- Optional polish if ROI is obvious

Avoid infinite refinement loops.

## Quality bar checklist
Before final delivery, ask:
- Is the hook immediate?
- Is the protagonist humiliation or pressure clear?
- Is the reversal satisfying?
- Are character motives legible?
- Does each scene change something?
- Is the ending sticky enough to pull the next episode?
