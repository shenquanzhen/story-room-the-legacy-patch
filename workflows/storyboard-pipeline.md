# storyboard-pipeline.md

## story-room storyboard workflow

Use storyboard generation after the script is usable.

## Phase 1 - Script lock check
Before storyboarding, confirm:
- episode beats are stable
- scene order is stable
- key dialogue is mostly stable
- hook and cliffhanger are fixed enough to visualize

If the script is still moving heavily, revise script first.

## Phase 2 - Parallel visual pass
Spawn in parallel:

### storyboard-agent
Produce:
- shot list by scene
- framing choices
- camera movement suggestions
- image hooks
- cliffhanger visual beat
- production-efficiency notes

### continuity-agent (optional)
Produce:
- prop continuity risks
- eyeline / movement continuity risks
- wardrobe / injury / location callback checks

## Phase 3 - Storyboard review
Spawn storyboard-review-agent with summaries from the script and storyboard branches.
Review must check:
- shot clarity
- emotional readability
- pacing rhythm
- coverage gaps
- production practicality
- whether the hook and cliffhanger land visually

## Phase 4 - Focused revision
If revise:
- send only must-fix items back to storyboard-agent
- if visual problems are caused by script issues, patch the script first, then update the storyboard
- do not fully redraw the board unless the scene design is broken

## Phase 5 - Final assembly
Deliver:
- locked script or current script basis
- storyboard shot list
- visual notes
- storyboard revision notes

## Quality bar checklist
Before final delivery, ask:
- Can each beat be understood without extra explanation?
- Does each important line have a strong image around it?
- Is coverage efficient instead of redundant?
- Is the hook visual in the first few shots?
- Is the final cliffhanger image sticky?
- Can this be shot practically for a short-drama workflow?
