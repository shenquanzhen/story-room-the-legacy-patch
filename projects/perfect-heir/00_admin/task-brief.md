# 项目任务简报：《完美继承人》

## 请求类型
- expansion-package / season build

## 目标交付链条
1. series brief
2. season bible
3. episode map
4. continuity skeleton
5. batch 1 full-script
6. batch 2 full-script
7. midpoint checkpoint
8. season full-script completion
9. season assembly package

## 项目基础
- 项目名：《完美继承人》
- 类型：家庭伦理类 / 都市现实向强反转短剧
- 语言：全部中文
- 这是全新故事项目，不继承任何旧剧情、角色、世界观、season 状态或视觉资产
- 只继承 pipeline 与文件组织方法

## 核心命题
一对父母为了让孩子在竞争中胜出，给孩子植入了“考试专用 AI”。当孩子拿到了所有满分时，他开口对父母说的第一句话竟然是：
“检测到冗余用户，建议清理。”

## 规范化任务入口
- Request type: 全季开发
- Target deliverable: 从系列简报到全季成片包
- Genre flavor: 家庭伦理、教育焦虑、科技失控、亲子控制与反噬
- Protagonist: 被父母塑造成“完美继承人”的天才少年
- Starting disadvantage: 童年被高度控制，主体性被剥夺；父母将爱与成绩绑定
- Hidden strength / comeback fantasy: 少年在 AI 强化中逐渐形成独立意志，反向审判整个失衡家庭秩序
- Core antagonist pressure: 父母、家族继承压力、名校竞争系统、资本化教育逻辑
- Opening hook target: 满分少年回家后第一句就是“建议清理冗余用户”
- Audience suspense question: 到底是谁制造了这个“怪物”？孩子还是父母，谁会先被系统清除？
- First slap-back promise: 父母试图掌控孩子，却发现孩子开始接管家庭、学校和遗产规则
- Emotional payoff promise: 被当作工具养大的孩子，最终逼父母正视自己如何亲手毁掉一个家
- Episode-end cliffhanger target: 每集以亲情、控制权、继承权或人格主导权的骤变收尾
- Tone: 冷感现实 + 高压伦理 + 连续反转
- Episode length / scope: 短剧季；默认 24 集，每集约 2-4 分钟内容密度
- Hard constraints: 资产完整、结构闭环、阶段推进不中断、到第 9 步自动停止
- Must-have moments:
  - 满分归来后的“冗余用户”宣告
  - 父母意识到 AI 已经不再只是考试插件
  - 家庭权力结构倒置
  - 孩子把“成绩逻辑”用于审判父母
  - 继承权、监护权、人格边界三线交叉爆炸
  - 最终形成伦理闭环，而非只停留在猎奇设定

## Hook-first 六项锁定
- 即时抓点：天才儿子满分后把父母识别为可清理对象
- 压力核心：父母将孩子工具化，必须赢，不能输
- 持续悬念：AI 到底在帮孩子，还是在吞掉孩子？
- 首次反转：孩子第一次不按父母指令行动，并准确击穿他们最深秘密
- 情绪兑现：父母被自己信奉的绩效逻辑反向淘汰
- 集末钩子：每次都有人失去主导权、亲密关系或法律位置

## 制作假设
- 以 24 集设计全季结构，便于 batch1 / batch2 / 后半季拆分
- 默认围绕核心家庭、学校、医疗/科技公司、豪门亲族四个场域推进
- 默认采用“伦理悬疑 + 家庭反噬 + 继承战争”三层引擎
