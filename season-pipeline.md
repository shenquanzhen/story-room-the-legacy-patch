# story-room season pipeline

This file describes the practical end-to-end production flow for a full 20-episode season.

---

## 1. Entry sequence

When the user requests a full season:
1. build series brief
2. build season hook spine
3. build 20-episode map
4. validate continuity skeleton
5. open batch 1 drafting

Do not draft episode scripts before the episode map exists.

---

## 2. Step-by-step season flow

## Step 1 - Series brief
Define:
- premise
- target audience fantasy
- humiliation engine
- comeback fantasy
- genre flavor
- season target length
- constraints

## Step 2 - Season hook spine
Define:
- opening season hook
- core season suspense question
- protagonist season reversal fantasy
- midpoint turn
- pre-finale collapse or trap
- final payoff image

## Step 3 - 20-episode map
For ep01-ep20, define:
- episode role
- episode hook
- episode pressure
- episode reversal
- episode cliffhanger
- continuity dependencies

## Step 4 - Continuity skeleton review
Check before scripting:
- reveal order
- relationship progression
- antagonist escalation
- payoff pacing
- callback opportunities

## Step 5 - Batch drafting
Draft in windows:
- ep01-ep05
- ep06-ep10
- ep11-ep15
- ep16-ep20

For each batch:
1. derive episode hook boards
2. derive episode outlines
3. derive script drafts
4. run episode reviews
5. patch local issues
6. run continuity pass across the batch
7. promote approved batch artifacts

## Step 6 - Batch checkpoint review
After each batch, review:
- continuity health
- escalation curve
- protagonist arc progression
- antagonist pressure consistency
- unresolved thread status

## Step 7 - Late-season integrity review
Before ep16-ep20 finalization, run special checks:
- does final payoff still exceed early payoff
- has hidden power been overspent
- do remaining turns still feel fresh
- does finale earn the season promise

## Step 8 - Final season assembly
Assemble:
- season bible
- episode scripts
- continuity summary
- optional storyboard package
- season review memo

---

## 3. Hard gating rules

### Gate 1
No batch drafting before episode map approval.

### Gate 2
No episode script promotion before episode review.

### Gate 3
No batch promotion before continuity pass.

### Gate 4
No final season promotion before late-season integrity review.

---

## 4. Recommended pacing logic

For a 20-episode short drama, use broad pacing like this:
- ep01-ep03: humiliation + hook + first proof of hidden force
- ep04-ep08: social reversals and widening conflict
- ep09-ep12: stronger counters, relationship turns, mid-season shock
- ep13-ep16: backlash, trap, betrayal, apparent loss
- ep17-ep20: comeback surge, reckoning, final public payoff

This is a default pattern, not a fixed law.

---

## 5. Failure containment rule

If one episode fails, do not restart the season.
Contain failure by asking:
- is this local to one episode
- is it local to one batch
- is it season-structural

Repair at the smallest valid scope.
