# story-room season orchestrator playbook

This file is the practical operating guide for running a full season pipeline.

---

## 1. First move on a season request

When a user requests a full season:
1. normalize the series brief
2. check whether season artifacts already exist
3. create or load season spine
4. create or load episode map
5. run continuity skeleton review
6. open the first drafting batch only after gates pass

Do not begin episode scripting before season structure exists.

---

## 2. Startup checklist for a season run

Before opening a season run, confirm:
- series brief exists or not
- season hook spine exists or not
- episode map exists or not
- continuity ledger exists or not
- reveal tracker exists or not
- callback tracker exists or not
- current approved batches exist or not
- stale batches exist or not

If any controlling season artifact is missing, create it before batch drafting.

---

## 3. Season opening sequence

### Step 1
Create series brief.

### Step 2
Create season bible / season spine.

### Step 3
Create 20-episode map.

### Step 4
Run season skeleton review.

### Step 5
If season skeleton ships, open batch 1.

---

## 4. Batch operation procedure

For each batch:
1. load latest approved season artifacts
2. load continuity state
3. derive episode hook boards for the batch
4. derive outlines and characters if needed
5. draft scripts
6. run per-episode reviews
7. patch local issues
8. run batch continuity review
9. promote approved batch scripts
10. update continuity artifacts
11. decide whether next batch can open

---

## 5. Checkpoint review procedure

After each batch, ask:
- is escalation still rising
- is payoff reserve still healthy
- are reveals still properly distributed
- is protagonist progression coherent
- is antagonist pressure still dangerous
- are unresolved threads under control

If not, patch the controlling season artifact before opening the next batch.

---

## 6. Season structural patch rule

If season spine or episode map changes:
- identify affected episodes or batches
- mark downstream artifacts stale
- block autonomous advancement into the next affected batch
- regenerate at the smallest valid season scope

---

## 7. Autonomous advance rule

Advance automatically to the next batch only when:
- current batch approved
- continuity updated
- checkpoint review passed
- no critical season contradiction exists

---

## 8. Season closeout

At the end of the season run, ensure:
- season bible approved
- episode scripts approved
- continuity ledger updated
- major reveals and callbacks accounted for
- final season delivery assembled
